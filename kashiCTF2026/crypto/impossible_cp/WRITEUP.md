# Impossible CP

**CTF**: kashiCTF 2026
**Category**: Crypto
**Points**: 347
**Flag**: `kashiCTF{d8b4ea850c6d4d34f74f66f60efd2904Dh7rXKTCzl}`

## TL;DR

Interactive problem derived from Codeforces 2156/D. Hidden array generated by MT19937 — recover 624 outputs via bit queries, crack the PRNG state with `randcrack`, predict A_n.

## Analysis

- Given: hidden array A of length n, query `? i x` returns 0 if `A_i & x == 0`, 1 otherwise
- Constraint: can only query indices 1..n-1, must find A_n
- Budget: 20n queries per test case
- Without permutation guarantee, A_n is information-theoretically impossible to deduce from A_1..A_{n-1}

**Key insight**: `strings` on the checker binary reveals it uses the Rust `mt19937` crate (v3.2.0). Array values are raw 32-bit MT19937 outputs, not bounded by n.

## Vulnerability

MT19937 is deterministic and trivially reversible from 624 consecutive 32-bit outputs. The problem gives us access to indices 1..n-1 (n ≥ 1000), so we can recover 624 values and predict all subsequent outputs.

## Exploitation

1. **Bit extraction**: For each of 624 elements, query all 32 bits (`? i 1`, `? i 2`, `? i 4`, ..., `? i 2^31`) — 19968 queries total, fits in 20n budget
2. **State recovery**: Feed 624 values to `randcrack` to recover full MT19937 internal state
3. **Prediction**: Generate forward to position n to predict A_n
4. **Multi-testcase**: MT state is continuous across test cases, so subsequent test cases need no queries

Threading used to pipeline queries (send in batches while reading responses) to avoid pipe deadlock and meet the 10-minute server timeout.

## Key Takeaways

- "Impossible" problems in CTFs often have a hidden assumption to exploit (here: weak PRNG)
- MT19937 is never cryptographically secure — 624 outputs = full state recovery
- Always `strings` the binary for implementation hints (crate names, library paths)
