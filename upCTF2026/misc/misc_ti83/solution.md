# PROG.8xp (TI-83+ Code Checker) — upCTF 2026

**Category:** MISC (Reversing / Z80 Assembly)
**Flag:** `upCTF{1F41L3DC4LCF0RTH1S}`

## TL;DR

Z80 assembly program for TI-83+ calculator that implements a "code checker" with anti-reversing protection: a validation routine encrypted with XOR whose key is derived from a CRC-8 accumulated by an ISR (timer interrupt). The key is deterministic (depends only on the number of characters, not the input) and can be obtained by brute force (1 byte) or direct calculation.

## Analysis

### Reconnaissance

```
$ file PROG.8xp
PROG.8xp: TI-83+ Graphing Calculator (assembly program)
```

`.8xp` format with header `**TI83F*`, comment `File generated by spasm`. The binary contains Z80 assembly that loads at `$9D95`.

Visible strings:
- `= CODE CHECKER =`
- `Enter code (CLEAR=reset):`
- `CORRECT! :)` / `WRONG. Try again.`

### Program Architecture

```
0x9D95: JP 0x9E57          ; entry → setup
0x9D98: [18 bytes]          ; expected data (encrypted)
0x9DAA: [57 bytes]          ; comparison routine (encrypted)
0x9DE5: strings             ; UI text
0x9E31: ISR                 ; interrupt service routine (CRC-8)
0x9E57: setup + input loop  ; initialization and main loop
0x9F18: validation handler  ; ENTER handler
0x9F8B: token validator     ; validates key range
0x9FA1: token→ASCII         ; converts token to char for display
```

### Input and Tokens

The program accepts two TI-83+ token ranges:

| Token range | Characters | ASCII mapping |
|-------------|-----------|-------------|
| `0x8E–0x97` | 0–9 | `0x30–0x39` |
| `0x9A–0xB3` | A–Z | `0x41–0x5A` |

Maximum 18 characters (`0x12`). CLEAR resets. ENTER validates.

### ISR: CRC-8 via Timer Interrupt

```z80
; ISR installed in IM 2, executes on each timer tick
; Only processes if flag (0x9872) is set AND char count > 0
ISR:
    LD A,(0x9872)     ; check flag
    JR Z, done        ; skip if flag=0
    XOR A
    LD (0x9872),A     ; clear flag (runs ONCE per keypress)
    LD A,(0x9DE3)     ; B = current char count
    JR Z, done
    LD B,A
    LD A,(0x8749)     ; load CRC state (init: 0xA5)
crc_loop:
    SRL A             ; shift right, bit 0 → carry
    JR NC, skip
    XOR 0xB8          ; CRC-8 polynomial feedback
skip:
    DJNZ crc_loop     ; loop B times
    LD (0x8749),A     ; store updated CRC
```

The flag is activated once per keypress → ISR executes **exactly once** per character, with `B = char_count` at that moment.

Total CRC iterations for 18 characters:

```
1 + 2 + 3 + ... + 18 = 18 × 19 / 2 = 171
```

### Validation (ENTER Handler)

```z80
validate:
    LD A,(0x9DE3)     ; char count
    CP 0x12           ; must be exactly 18
    JR NZ, wrong

    ; Decrypt comparison routine
    LD A,(0x8749)     ; CRC result
    XOR 0x17          ; derive XOR key
    LD C,A
    LD HL,0x9DAA      ; encrypted routine (57 bytes)
    LD DE,0x8710      ; destination
    LD B,0x39         ; 57 bytes
decrypt_loop:
    LD A,(HL)
    XOR C             ; single-byte XOR decrypt
    LD (DE),A
    INC HL / INC DE
    DJNZ decrypt_loop

    ; Call decrypted routine
    LD HL,0x9D98      ; expected data (18 bytes)
    LD DE,0x86EC      ; user input buffer
    LD B,0x12         ; 18 chars
    LD C,0xA5         ; keystream seed
    CALL 0x8710       ; compare → carry=wrong, no carry=correct
```

### Decrypted Comparison Routine

With XOR key `0xA7`, the decrypted routine does:

```z80
; Phase 1: Generate expected tokens via CRC-8 keystream
    LD DE,0x86FE      ; temp buffer
    ; C = 0xA5 (seed)
transform:
    SRL C             ; CRC-8 step
    JR NC, skip
    XOR C, 0xB8
skip:
    LD A,(HL)         ; load encrypted expected byte
    XOR C             ; decrypt with keystream
    LD (DE),A         ; store plaintext token
    INC HL / INC DE
    DJNZ transform

; Phase 2: Compare input vs expected
    LD HL,0x86FE      ; decrypted expected
    POP DE            ; user input (0x86EC)
    LD B,0x12
compare:
    LD A,(DE)
    CP (HL)           ; byte-by-byte compare
    JR NZ, wrong      ; mismatch → SCF; RET
    INC HL / INC DE
    DJNZ compare
    AND A             ; match → clear carry; RET
```

## Exploit

### XOR Key Calculation

CRC-8 with polynomial `0xB8`, initial value `0xA5`, 171 iterations:

```
CRC(171 iters) = 0xB0
XOR key = 0xB0 ⊕ 0x17 = 0xA7
```

This can also be obtained by brute force: try 256 XOR keys, the one that produces valid Z80 (ends with `RET`, contains `LD A,(HL)`, `DJNZ`, etc.) is `0xA7`.

### Decrypting the Expected Data

Apply the CRC-8 keystream (seed `0xA5`) to the 18 bytes at `0x9D98`:

```
Encrypted: 65 ea 10 ce 3d dd bb 8f 23 45 ec a7 92 a5 aa 1a 6a 66
Tokens:    8F 9F 92 8F A5 91 9D 9C 92 A5 9C 9F 8E AB AD A1 8F AC
ASCII:      1  F  4  1  L  3  D  C  4  L  C  F  0  R  T  H  1  S
```

### solve.py

```python
#!/usr/bin/env python3
data = open("PROG.8xp", "rb").read()

bb6d = data.find(b'\xbb\x6d')
base, code_start = 0x9D95, bb6d + 2

def off(addr): return code_start + (addr - base)

# CRC-8: init 0xA5, poly 0xB8, 171 iterations → XOR key
crc = 0xA5
for _ in range(171):
    carry = crc & 1; crc >>= 1
    if carry: crc ^= 0xB8
xor_key = crc ^ 0x17  # 0xA7

# Decrypt expected data with CRC-8 keystream
expected_enc = data[off(0x9D98):off(0x9D98) + 18]
c = 0xA5
tokens = []
for b in expected_enc:
    carry = c & 1; c >>= 1
    if carry: c ^= 0xB8
    tokens.append(b ^ c)

# Token → ASCII
code = ''.join(
    chr(t - 0x8E + 0x30) if 0x8E <= t <= 0x97 else
    chr(t - 0x9A + 0x41) for t in tokens
)
print(f"upCTF{{{code}}}")
```

## Key Lessons

1. **8xp format**: TI-83+ files contain Z80 with a specific header, code loaded at `$9D95`, marker `BB 6D` for assembly
2. **ISR as anti-RE mechanism**: using timer interrupts to compute a state (CRC) that conditions the validation adds a layer of indirection that complicates static analysis
3. **Hidden determinism**: although the ISR appears to depend on timing, the "once per keypress" flag makes it completely deterministic — 171 fixed iterations
4. **Single-byte XOR key**: always brute-forceable (256 options), validating legitimate Z80 in the decrypted output reduces to a single solution
5. **Dual CRC-8**: the same algorithm (poly `0xB8`) is used for both the decryption key AND the expected data keystream, but with different seeds/iterations
